I would like to suggest that for OpenViewer we should concentrate on getting a well defined setup on our computers, be they Windows or Linux. Then next a well defined recipe for compiling with no errors and minimal warnings. Then third a well defined test on a common region or regions that allows anyone to test OpenViewer.

I would suggest that the first one be defined on our wiki so we can say:

"See wiki page X for the setup required for your computer to compile successfully OpenViewer".

Then the second one be defined so we can say:

"See wiki page Y and checkout our svn, runprebuild.bat, build, run and it should run with no errors"

The the third one be defined so we can say:

"Logon to standaloneA or GridG to the region framelwanker and you should see terrain, prims, walk, chat and logoff with no errors"

Charles
