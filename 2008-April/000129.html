(Some opinions and not necessarily in the order presented)

First, wrt "User should be able to type in action text": 
The client should change "/me" to the speaking user's name.  The user shouldnt 
have to type in action text, the client should just handle it.  

Second, wrt the about screens: 
Can we load that text from a file instead of embedding it inside the glade 
files?

"Enter in a URL for us to try to extrapolate a grid for them to connect to":
So then it would basically brute-force a loginuri? Like, if a user entered 
"osgrid.org", would it just try logging in using "http://www.osgrid.org:8002"?

The address bar controls are rather unintuitive.  Those buttons should 
probably go into menu options instead, and be replaced with a "Go!" button on 
the right side of the address bar.  If I want my browsing history, I'll go 
to "History", if I want my favorites, I'll go to "Favorites" just like every 
other browser since NCSA Mosaic.  Also, connect/disconnect is rather weighty 
to put on the address bar, since personally it takes about 20 seconds to log 
in to any grid.  It's comparable to re-dialing on a dialup modem, and I dont 
need to accidentally click that while I'm browsing.  If I want to connect, 
I'll type it in or select it from a list and hit enter or click the "go" 
button.  If I want to disconnect, I'll close the tab (or the whole client), 
or if it's there and I really want to, a "logout" selection from a mennu 
somewhere.

Also, in the "Accounts" dialog, instead of "Add" perhaps consider naming that 
button "New".
