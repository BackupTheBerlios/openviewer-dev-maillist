I think that the Testing UI module could take the form of common/easy 
scripting interface for writing scripts to control the client. IronPython 
maybe? XML?

Also, once this is done, a web-interface for it becomes trivial, if we can 
specify input from a module as well as from various UI elements (keyboards, 
mice, etc)

On Friday 07 March 2008 1:30:20 pm Daedius wrote:
> I wanted to put this idea on the mailing list so I do not forget it.
> When I came to openviewer,  I  initially did not plan to be working on
> 3d or GTK# interfaces.   I love automated testing and was the first to
> integrate NUnit into the OpenSim project.  My vision was this:
>
> "Functional testing of metaverses is not enough.   If I could simulate
> a person using a real viewer.  I might be able to get better results
> in automated realtime testing of grids"
>
> A viewer doesnt have to be controlled by a user.  It can be controlled
> by a robot ;)  or even replayed.   I figure a Testing UI module down
> the line will be very helpful to the opensim community as a whole.
> Any thoughts or ideas?  Praise or condemnation?
