On Friday 07 March 2008 3:07:56 pm Daedius wrote:
> I like your idea.   My preference is to have nunit tests written.   In
> my ideal world, it would be awesome if we could write unit tests that
> A) intantiate grids ( on multiple machines even possibly?) B) load up
> our tester UI and connect to them , then C) run scripted tests like a
> normal user would.    We might be able to have a collection of scripts
> written in iron python ( or any other language ), have mono compile
> them on the fly, and run them within the unit test.

Yeah, I meant that developers would have their unit tests, then the scripting 
interface for the client would enable a user to basically click a button and 
have the agent the client was connected to to be moved automatically, taking 
actions like testing inventory, movement, chat, etc.  (Then, not only would 
it be an automated testing interface, but a scripted client for "free" :P)

>
> I during january I was writting a distributed crossplatform build
> system that ran tests through master slave architecture, hand off jobs
> that were entirely specified trough nant.  My goal was to create a
> testing environment for the diverse crossplatform machines opensim was
> meant to run on.  The architecture is very similar to BuildBot if that
> means anything to you.  I even was able to get realtime results of
> builds happening across seperate machines =)  all the test/build
> results were available via webservice, and then simply accessed by
> PHP.  It was teh awesome.

Yes, I saw that, it was quite interesting :D
Speaking of tests, how is openviewer unittest-wise?

>
> I'm curious on how IronPython works as a scripting language?  Is it
> very crossplatform at this point in time?

I'd like to say it's cross platform, but the ipy.exe binary has an issue on 
Ubuntu Gutsy where it doesnt display any output to consoles, but will write 
to stdout just fine.  In other words, it's crossplatform enough to not worry 
about it.  (Personally I think IronPython is blasphemy, but I'd rather see a 
scriptable interface in a Python language than in VB.NET :P)
