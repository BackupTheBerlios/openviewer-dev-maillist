The problem lies in our base using libsecondlife, which is in C#.  if
we go with python, then we have a bit of a problem.  We would then
have to write our own networking code from scratch.  I would rather
not re-invent the wheel.  If we can use any language, why not go with
regular Ogre in C++ for example?

There are other reasons for choosing c# though.  CLR makes it easy to
port existing projects to it.  It is a managed language which relieves
us from a lot of work, and headache in the future, many of the
developers in the community prefer it over other languages, to name a
few.

Regards,
Adam J
